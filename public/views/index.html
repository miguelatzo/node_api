API

Autenticación

POST /auth/otorgar-token/

Cuerpo de la petición POST

  • correo: el correo del usuario de la API
  • contrasenia: la contraseña del usuario de API

Respuesta del servidor

  • el payload de la respuesta del servidor es un JSON de la manera {"token": timestamp}
  • Es necesario añadir la cabecera Token: {token} en cada petición posterior

Endpoints

GET /operaciones/suma/?n1={$1}&n2={$2}...

parámetros de consulta tradicionales

  • n1, n2, n3... : son identificadores válidos
  • $1, $2, $3... : son números reales considerados válidos en el lenguaje JavaScript

GET /operaciones/suma/$1,$2,$3...

Un parámetro personalizado

  • $1, $2, $3... : son números reales considerados válidos en el lenguaje JavaScript

GET /operaciones/multiplicacion/?n1={$1}&n2={$2}...

parámetros de consulta tradicionales

  • n1, n2, n3... : son identificadores válidos
  • $1, $2, $3... : son números reales considerados válidos en el lenguaje JavaScript

GET /operaciones/multiplicacion/$1,$2,$3...

Un parámetro personalizado

  • $1, $2, $3... : son números reales considerados válidos en el lenguaje JavaScript

GET /operaciones/raices/?a={$1}&b={$2}&c={$2}

parámetros de consulta tradicionales

  • a, b y c: deben escribirse en ese orden
  • $1, $2, $3... : son números reales considerados válidos en el lenguaje JavaScript

GET /operaciones/raices/$1,$2,$3...

Un parámetro personalizado

  • $1, $2, $3: representan a, b y c y son números reales considerados válidos en el lenguaje JavaScript
